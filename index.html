동작 원리 (How it Works)

상단의 실행 버튼을 누르면 Web Worker 백그라운드 환경에서 빌드가 수행됩니다:

  • 가상 파일 시스템 (MemFS): C++ 소스코드를 메모리 속 가상 디렉토리 구조에 임시 파일로 저장합니다.
  • 컴파일 (Clang): WebAssembly 기반 Clang 8.0.1 컴파일러가 가상 파일로부터 ELF 목적 파일(.o)을 빌드합니다. 대상 아키텍처는 wasm32-wasi입니다.
  • 링킹 (LLD): 목적 파일을 가상 POSIX 헤더/표준 라이브러리(libc.a, libc++.a) 및 Canvas API 브릿지와 연결하여, 독립적으로 구동되는 WebAssembly 바이너리(.wasm)를 구성합니다.
  • 가상 머신 실행: WASI API 인터페이스 위에서 모듈을 가동하고, 터미널 입출력 스트림을 웹 터미널 화면으로 실시간 연동합니다.

Canvas 라이브러리 그래픽 인터페이스

그래픽 화면을 그리려면 소스코드에 #include <canvas_main.h>를 삽입하고 다음 함수를 필수로 정의해야 합니다:

  • void setup(): 프로그램의 초기 상태(캔버스 해상도 등)를 1회 셋업합니다.
  • void loop(double time, double elapsed): 매 1/60초(60FPS) 주기마다 실시간 렌더링 루프를 돕니다.
